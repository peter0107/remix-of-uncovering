# 미션 진행 화면 모바일 최적화 플랜

대상 파일: `src/routes/mission.$orderId.tsx`, `src/components/MaterialBlocksView.tsx`

현재 모바일(390px)에서 발견되는 문제와 개선안입니다.

## 현재 문제점

1. **우측 사이드바(360px 고정)** 가 모바일에서는 하단에 길게 쌓여서 "제출 영역"이 한참 스크롤해야 보임 — 제출 버튼 접근성 낮음
2. **헤더 패딩(px-6 py-3)** 과 본문 패딩이 모바일에 과도함
3. **제목(text-4xl)** 이 모바일에서 너무 큼
4. **제공 자료 카드 `p-6`** 가 모바일에서 좌우 여백 낭비
5. **표(Table) 블록** 은 `overflow-x-auto`만 있어 스와이프 가능하지만 컬럼이 많으면 가독성↓ — 스와이프 힌트 없음
6. **이미지 max-h-[480px]** — 모바일에서도 그대로라 화면을 거의 다 차지
7. **Tabs(`전체 보기/화면/표/기타`)** — 라벨 길어지면 줄바꿈/잘림 가능
8. **드롭존 + 제출 항목 아코디언** 이 항상 펼쳐진 상태로 길게 노출 → 제출 버튼 도달까지 스크롤 길어짐

## 개선 방향

### A. 레이아웃 (mission.$orderId.tsx)
- 본문 패딩 반응형: `px-4 py-5 md:px-6 md:py-8`
- 헤더 패딩 축소: 모바일 `px-4`
- 제목 크기: `text-2xl md:text-3xl lg:text-4xl`
- 사이드바를 모바일에서 **하단 sticky 제출 바**로 분리
  - 데스크톱: 기존 우측 360px 컬럼 유지
  - 모바일: "리포트에 포함할 항목" 카드는 본문 흐름에 인라인 배치, "최종 제출" 카드는 **fixed bottom bar** 형태로 화면 하단 고정 (파일명 + 제출 버튼)
  - bottom bar 아래 콘텐츠가 가리지 않도록 본문에 `pb-28 lg:pb-8` 적용

### B. 제공 자료 영역
- 카드 패딩 반응형: `p-4 md:p-6`
- Tabs를 모바일에서 가로 스크롤 가능하게 (`overflow-x-auto`, `whitespace-nowrap`)
- 이미지: `max-h-[320px] md:max-h-[480px]`, 탭 또는 클릭으로 풀스크린 보기(선택)
- 표: 컬럼 2개 초과 시 스와이프 안내 텍스트("← 좌우로 스크롤 →") 모바일에서만 표시, 첫 컬럼 sticky 처리

### C. 제출 UX
- 모바일 bottom bar 구성:
  - 좌측: 파일 상태(미업로드 / 파일명 truncate)
  - 우측: 업로드/제출 버튼 (파일 없으면 "업로드", 있으면 "제출하기")
- 드롭존은 탭하면 파일 선택만 동작(드래그는 모바일에서 의미 없음) — 콤팩트 버전(`py-6`)으로 표시
- "리포트에 포함할 항목" 아코디언은 모바일에서 기본 접힘 상태로 시작

### D. 기타 디테일
- 상단 브래드크럼 텍스트 잘림 방지: `truncate`
- Badge들 `flex-wrap gap-1.5`로 줄바꿈 안정화
- BriefCard 아이콘 + 본문 stacked on mobile은 유지하되 패딩 축소

## 구현 단계

1. `mission.$orderId.tsx`
   - 헤더/본문 패딩, 제목 크기 반응형 조정
   - 그리드를 `lg:grid-cols-[...]`로 모바일에서 single column 유지(이미 그러함) + 사이드바 순서 재배치
   - 모바일 전용 `MobileSubmitBar` 컴포넌트 추가 (fixed bottom, `lg:hidden`)
   - 기존 사이드바 제출 카드는 `hidden lg:block`로 가리고, 모바일에서는 인라인 콤팩트 드롭존 + 하단 bar
   - 본문에 `pb-28 lg:pb-8`
2. `MaterialBlocksView.tsx`
   - 카드 패딩 반응형
   - 이미지 max-h 반응형
   - 표에 모바일 스와이프 힌트 및 스크롤 컨테이너 스타일 보강
3. 동작 검증: 390x844 viewport로 미리보기 확인 (스크린샷)
