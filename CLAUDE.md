# CLAUDE.md

Claude Code가 세션 시작 시 자동으로 읽는 파일. 이 프로젝트에서 작업할 때 항상 적용.

## 프로젝트 개요
Beginner — Forage 모델의 한국형 적응. 구직 대학생이 지원 전에 실제
업무 과제를 시뮬레이션으로 경험하고, 기업은 이를 통해 고관심 지원자를 발견하는
양면 플랫폼. 자기보고(성격검사)가 아니라 행동(실제 과제 수행)으로 직무 적합도를
드러낸다는 철학.

- 배경·의사결정 전체 맥락: @docs/beginner_project_brief.md
- DB 스키마: @docs/beginner_schema.sql

## 절대 규칙 (변경 없이 항상 적용)

1. **사이트는 3개로 완전히 분리, 절대 하나로 합치지 않는다.**
   - 구직자 사이트 (beginner.today) — 시뮬레이션 수행
   - 기업 사이트 (Beginner biz) — 고유코드 로그인, 지원자 열람 전용,
     구직자 사이트와 별개 도메인/앱
   - groupby — 완전 외부 채용사이트. '지원하기' 버튼의 딥링크 대상일 뿐 우리 DB와 무관

2. **답안 공개는 A안(원문 전송) + 동의 기반.** 두 동의를 절대 하나로 합치지 않는다.
   - `discovery_consent`: 기업이 이메일로 제안 보내도 되는지 (온보딩 마지막 단계)
   - `answer_transmission_consent`: 이 답안을 기업에 전송해도 되는지 (제출 시)
   - `answer_transmission_consent = true`인 제출물만 기업 사이트에 노출

3. **온보딩은 성향검사가 아니다.** "이건 테스트가 아니에요" 류의 카피 금지.
   목적은 성향 파악이 아니라 "어떤 직무 시뮬레이션을 받고 싶은지" 좁히기.
   Forage 사전질문 흐름 참고 (학력 → 관심직무 → 관심기업 → 근무선호 → discovery 동의).

4. **추천 로직은 룰 기반만 사용한다.** IRT, Contextual Bandit 등 적응형 알고리즘은
   이 프로젝트에서 사용하지 않는다 (다른 서비스의 인프라이며 여기선 불필요).

5. **프로필(이력서)은 최대한 심플하게 유지한다.** 자동이력(완료 시뮬 목록) +
   한줄소개 + 외부링크(GitHub/포트폴리오/LinkedIn) + 프로필 사진 + 온보딩 답변
   (학력/전공/관심직무/관심기업/근무선호/discovery 동의)을 프로필 화면에서
   직접 열람·수정하는 것까지만. 11축 역량 배지/레이더차트, PDF 내보내기 같은
   부가 기능은 넣지 않는다 (기능 과다로 인한 혼란 방지가 목적, MVP 이후로 미룸).


## DB 마이그레이션 워크플로우
이 프로젝트는 Lovable 기반 Supabase라 Claude Code가 `supabase db push`로 직접 반영할
수 없다. **DB 스키마를 바꿔야 할 때마다:**
1. `supabase/migrations/`에 새 마이그레이션 SQL 파일을 작성한다 (기존 네이밍 규칙 따름).
2. 그 파일의 SQL 전체 내용을 채팅 메시지에도 코드블록으로 그대로 올려서, 사용자가
   Lovable SQL Editor에 바로 복사해 넣을 수 있게 한다 (파일 작성만 하고 끝내지 않는다).

## 기술 스택
Next.js 14 + Tailwind CSS + Anthropic SDK + Supabase + Recharts, Vercel 배포.
React Native/Expo는 모바일 앱용 (MVP 웹 이후 순서).
